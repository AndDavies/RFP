## What the Next Session Should Focus On

The next development session should focus on implementing the ingestion processing worker, which will complete the end-to-end data pipeline between validated ingestion records and the production opportunity tables.

The current ingestion workflow successfully stores scraped opportunities in the rfp_ingestion_raw table and allows administrators to review and approve records through the /admin/ingestion interface. Approved records are inserted into the rfp_ingestion_queue, but there is not yet a process that converts these queued records into structured production data.

The objective of the next session is to build a processing component that reads validated records from rfp_ingestion_queue, parses the payload, and inserts the appropriate fields into the production tables.

The worker should perform the following responsibilities.

Retrieve records from rfp_ingestion_queue where the associated raw record has already been validated.

Parse the parsed_data JSON payload into structured fields expected by the production schema.

Insert the opportunity into the rfps table with normalized fields including title, agency, description, deadline, and source metadata.

Create related records when applicable. This may include inserting document links into rfp_documents or storing raw payload references for traceability.

Prevent duplicate records by checking existing entries using the source_id and source_external_id uniqueness constraint.

Mark processed queue records so they are not processed again. This may involve adding a processed flag or updating the queue record status.

Log processing actions to support debugging and ingestion auditability.

Once this processing worker is implemented, the ingestion pipeline will function end-to-end.

Scraping agents will submit raw opportunity payloads to the ingestion endpoint. These records will appear in the admin ingestion review interface. Approved opportunities will be moved to the ingestion queue, and the processing worker will convert them into structured RFP records that populate the SaaS dashboard.

After the processing worker is complete and verified, the following session should begin implementing the first scraping agent that submits opportunities to the ingestion endpoint.
